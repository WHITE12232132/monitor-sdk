# Monitor SDK 需求文档

## 项目简介

针对 React / Vue 应用的轻量级前端监控 SDK，具备**错误采集、性能监测、用户行为追踪、录屏回放**四大能力，助力线上问题快速定位。

## 工作职责

- SDK 架构设计
- 错误 / 性能 / 行为多维度采集
- 数据上报策略设计
- 录屏回放功能开发

---

## 需求清单

### 1. 架构设计

| 需求 | 说明 | 状态 |
|------|------|------|
| 模块化架构 | 错误、性能、行为、录屏回放四大模块独立 | ✅ |
| EventBus 解耦 | 模块间通过 EventBus 通信，互不感知 | ✅ |
| 插件化按需启用 | 通过配置控制模块开关，不用的不加载 | ❌ |
| SDK 自身循环上报防护 | 防止 SDK 自己发出的请求被采集 | ❌ |

### 2. 错误采集

| 需求 | 说明 | 状态 |
|------|------|------|
| 全局 JS 错误 | `window.onerror` / `error` 事件 | ✅ |
| Promise 拒绝 | `unhandledrejection` 事件 | ✅ |
| 资源加载失败 | 资源标签的 error 事件 | ✅ |
| Map 时间戳去重 | 同一错误在时间窗口内不重复上报 | ❌ |
| 批量上报策略 | 多条错误合并为一次请求发送 | ❌ |

### 3. 性能监测

| 需求 | 说明 | 状态 |
|------|------|------|
| LCP（最大内容绘制） | PerformanceObserver + largest-contentful-paint | ✅ |
| FCP（首次内容绘制） | PerformanceObserver + first-contentful-paint | ✅ |
| CLS（累计布局偏移） | PerformanceObserver + layout-shift | ❌ |
| 请求拦截链路追踪 | 劫持 fetch / XHR，采集接口耗时和状态码 | ❌ |

### 4. 行为追踪

| 需求 | 说明 | 状态 |
|------|------|------|
| 点击事件追踪 | 监听 click，记录 tagName + innerText | ✅ |
| 输入事件追踪 | 监听 input，1 秒节流，最多 10 条面包屑 | ✅ |
| 面包屑 FIFO | 超出上限丢弃旧数据 | ✅ |

### 5. 录屏回放

| 需求 | 说明 | 状态 |
|------|------|------|
| 增量录制 | 只录制 DOM 变更，不重复录完整快照 | ❌ |
| 输入脱敏 | 用户输入的敏感信息（密码、手机号等）不录制 | ❌ |

### 6. 数据上报

| 需求 | 说明 | 状态 |
|------|------|------|
| sendBeacon 优先 | 页面关闭时也能发送 | ✅ |
| fetch 降级 | 不支持 sendBeacon 时用 fetch + keepalive | ✅ |
| SDK 自身请求过滤 | 上报请求不被自身采集（循环防护） | ❌ |

## 实现优先级

| 优先级 | 模块 | 内容 |
|--------|------|------|
| P0 | 错误采集 | Map 时间戳去重 |
| P0 | 数据上报 | SDK 自身循环上报防护 |
| P1 | 错误采集 | 批量上报策略 |
| P1 | 性能监测 | CLS 指标采集 |
| P2 | 性能监测 | 请求拦截链路追踪 |
| P2 | 录屏回放 | 增量录制 + 输入脱敏 |
| P3 | 架构 | 插件化按需启用 |
